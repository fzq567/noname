## 一、现状分析

### 1.1 当前目录结构问题

```
蒸蒸日上/
├── extension.js (336.8KB)     # 所有代码在一个文件，难以维护
├── extension.js.bak           # 备份文件，不应提交
├── info.json
├── LICENSE
├── package.js
├── CompareCode.java           # Java代码，不应在JS扩展中
├── 20+张ex_xxx.jpg图片        # 散落在根目录
├── 多个.md文档                # 文档和代码混杂
└── noname教程/                # 22个教程文件
```

### 1.2 图片引用格式

extension.js 中使用的是无名杀扩展资源路径格式：

```javascript
// 武将图片
["ext:蒸蒸日上/ex_buqu.jpg", ...]

// 阵亡音频
["die:ext:蒸蒸日上/audio/die/ex_buqu.mp3", ...]

// 技能音频
audio: "ext:蒸蒸日上:2"
```

**关键发现**：`ext:蒸蒸日上/` 表示扩展根目录，图片迁移到子目录后，需要改为 `ext:蒸蒸日上/image/character/ex_xxx.jpg`

### 阶段2：extension.js 模块化拆分（中风险）

**目标**：将大文件拆分为多个模块

#### 步骤2.1：分析 extension.js 结构

先阅读 extension.js，了解其结构并整理出分析报告：

- precontent 部分
- content 部分
- arenaReady 部分
- 武将定义部分
- 技能定义部分

结合项目代码确保分析结果准确无误后才能开始执行下一步。

#### 步骤2.2：创建 src 目录结构

```
src/
├── main/
│   ├── precontent.js
│   ├── content.js
│   └── arenaReady.js
├── character/
│   ├── index.js      # 导出所有武将
│   └── 各武将文件.js
├── skill/
│   └── index.js      # 导出所有技能
└── lib/
    └── utils.js      # 工具函数
```

#### 步骤2.3：逐步拆分

**建议顺序**（从低风险到高风险）：

1. 先拆分工具函数到 `src/lib/utils.js`
2. 再拆分武将定义到 `src/character/`
3. 最后拆分 main 逻辑

#### 步骤2.4：新的 extension.js

```javascript
import { precontent } from './src/main/precontent.js';
import { content } from './src/main/content.js';
import { arenaReady } from './src/main/arenaReady.js';
import { characters } from './src/character/index.js';

export default {
    name: '蒸蒸日上',
    precontent,
    content,
    arenaReady,
    character: {
        character: characters,
        // ...其他配置
    },
    // ...
};
```

**风险**：中  
**可能问题**：模块导入导出错误、变量作用域问题  
**验证方式**：逐个功能测试
